PRODUCT REQUIREMENTS DOCUMENT (PRD)
Awestar.communities – Unified Sports Coordination Platform
Version: 1.0
Timeline: 5 Weeks
Support: 6 Months Post-Launch
Platform Type: Web Application (Responsive – Mobile First)

1. Product Vision
To build a structured, monetizable, multi-city sports coordination ecosystem that replaces fragmented WhatsApp-based game management with a centralized platform supporting:
    • Game bookings
    • Coaching batches
    • Tournaments & events
    • Sports eCommerce
    • Structured engagement feed
    • Commission-based revenue

2. Product Goals
Primary Goals
    • Centralize all city-level game activity
    • Enable frictionless game booking via payment
    • Introduce commission-based revenue model
    • Provide structured tools for coaches
    • Enable tournament organization
    • Launch in 5 weeks
Secondary Goals
    • Improve liquidity (game fill rate)
    • Increase repeat participation
    • Enable long-term data ownership
    • Prepare for multi-city scalability

3. User Roles
    1. Player (default role)
    2. Game Creator (Player privilege)
    3. Coach
    4. Admin

4. High-Level Modules
    1. Authentication & User Management
    2. City & Area Infrastructure
    3. Game Marketplace
    4. Coaching Module
    5. Tournament & Event Module
    6. Structured Activity Feed
    7. eCommerce Module
    8. Admin Dashboard

    10. Payment & Commission Engine

    • Basic profile fields:

        ◦ City
        ◦ Area
        ◦ Preferred sports
Profile Management
    • Edit profile

    • View:
        ◦ Games attended
        ◦ Upcoming games
        ◦ Interested games
        ◦ Coaching enrollments
        ◦ Tournament participation
